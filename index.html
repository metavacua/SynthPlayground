AI Agent Self-Improving Codebase

An experimental environment for a self-correcting AI agent that autonomously manages and evolves its own code and governing protocols.

Overview

This repository represents a pioneering research effort into the future of AI-driven software development. It serves as a live experiment for an AI agent designed to be self-correcting and self-improving. The primary goal is to create an agent that can autonomously manage, refactor, and enhance its own codebase and the formal protocols that govern its behavior.

This is not a conventional software project but a research platform exploring the boundaries of agentic systems and the co-evolution of AI agents and the codebases they inhabit.

Core Concepts

The project is built on a foundation of cutting-edge ideas that redefine the relationship between code, documentation, and the agents that interact with them.

The Provable Repository

Instead of being a simple collection of text files, this repository is architected as a formal, mathematical proof. The directory structure and build process are designed to be logically verifiable. In this paradigm:

  • Documentation is the Specification: A component's README.md file acts as a formal contract or "type" that specifies what it does.
  • Code is the Proof: The source code provides the constructive proof that the specification is met.
  • The Build is the Verification: A successful build is a formal, automated verification that the proof (code) matches the specification (documentation). This makes "documentation drift" impossible by design.

The Logically-Grounded Agent

To interact with a provable repository, the AI agent's own cognitive architecture is grounded in the same logical principles. Its actions are not based on heuristics alone, but on verifiable transactions. This approach aims to create a more reliable, predictable, and secure agent, architecturally incapable of performing actions that would leave the system in an inconsistent state.

Key Modules

The repository is organized into several key modules, each representing a core component of the project's architecture. Here are a few highlights:

Custom Languages: aura_lang/ and lfi_ill/

A unique feature of this project is its use of custom-designed languages for defining agent behavior and logical proofs:

  • aura_lang: A simple, C-style scripting language used for writing high-level agent commands and automation scripts.
  • lfi_ill: An implementation of "Paraconsistent Light Linear Logic," a research-level formal logic for reasoning under uncertainty and resource constraints. This is a cornerstone of the "Provable Repository" concept.

Agent Tooling: tooling/

This is the heart of the agent's capabilities. It contains a vast collection of Python scripts that the agent can invoke to perform its tasks. This includes tools for:

  • Building and compiling the repository's various artifacts (builder.py).
  • Parsing and interpreting the custom languages (aura_executor.py, plllu_runner.py).
  • Analyzing and refactoring code (refactor.py, symbol_extractor.py).
  • Managing the agent's own self-improvement and self-correction loops (self_correction_orchestrator.py).

You can explore the full extent of the agent's capabilities in the tooling/ directory.

Formal Logic System: logic_system/

This module provides the concrete implementation of the formal logic systems described in the project's research papers. It contains Python classes for representing logical formulas, sequents, and proofs, and it includes interpreters for various logics. This is where the high-level theory of the project is translated into executable code.

Getting Started for Developers

Interacting with this repository requires a different approach than a typical software project. The AGENTS.md file in the root directory serves as the primary entry point and "constitution" for the AI agent.

The project uses a unified build system driven by tooling/builder.py. To see available build targets:

python3 tooling/builder.py --list

To run a specific build target, such as compiling the protocols:

python3 tooling/builder.py --target protocols

For detailed contribution guidelines and development setup, please refer to the README.md and CONTRIBUTING.md files.
